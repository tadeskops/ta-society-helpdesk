Access lists

Society uses a strict linear hierarchy — Admin > Chairman > Secretary > Treasurer > Committee > Contributor > Manager > Resident. Each role may edit the roster of any tier strictly below their own; Admin edits every list (including the Admin list itself, subject to a one-admin-minimum guard). Lists you are not allowed to edit are shown but disabled.

Admins (top of hierarchy · at least one)
Chairman Full view + all treasury actions (inherits every tier below).
Secretary Full view + all treasury actions (above Treasurer in the hierarchy).
Treasurer Full view + all actions (approve / reject / pay / record expense).
Committee Triage + issue-body edits; treasury only via grandfather clause.
Contributor (new · below Committee) Narrow revocable grant above Manager. No treasury access.
Managers Assign / resolve / photo; treasury per-flag opt-in.
Grandfather clause active. None of Chairman / Secretary / Treasurer lists are seeded yet, so the legacy Committee+Admin gate is still granting access to the treasury dashboard. Seed at least one of the three to lock down access to the new capability model. Once seeded, plain committee members will no longer see the ledger unless they are also listed above.

Treasury & Reimbursements

All treasury feature flags, tunables, and the private receipt-storage path template live here in one place. Receipts (proofs, cash slips, expense bills) are uploaded to a private GitHub repo — identity of that repo is set by the worker environment variables GH_TREASURY_OWNER, GH_TREASURY_REPO and GH_TREASURY_BRANCH (with token GITHUB_TREASURY_TOKEN). The path within that repo is a template — placeholders: {yearMonth}, {kind} (proof|payment|receipt), {id} (RMB-* or EXP-*), {seq}, {name}.

Feature flags

Flag delegation (admin only)

Delegate one or more feature flags to a non-admin tier. That tier and every tier above it can then flip the flag from their own settings page without needing full admin write access. Admins can always toggle any flag. Leave a flag at — not delegated to keep it admin-only.

Booking receipt archive

When a booking is confirmed the worker composes a PDF receipt and pushes it to the private receipts repository. Only Society Managers, Committee and Admins can view archived receipts — residents do not see them. All paths are templates; available placeholders: {facilityCode} {facilityCodeLower} {facilityId} {id} {year} {month} {day} {yearMonth} {period} {periodKey}.
